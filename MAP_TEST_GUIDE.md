# 地图生成器测试指南

## 修复内容总结

### 问题 1: 地名重复
**原因**: 使用 Perlin 噪声生成地名索引时，所有城市都返回相同的地名
**解决方案**: 使用独立的 Mulberry32 随机数生成器，为每个城市生成不同的随机索引

### 问题 2: 地图频闪
**原因**: 多个 useEffect 互相触发，导致无限重绘循环
**解决方案**: 
- 使用 `initializedRef` 标记初始化状态
- 使用 `lastDrawnSeedRef` 跟踪已绘制的 seed
- 分离地图绘制和城市标记绘制逻辑
- 移除不必要的依赖项

### 问题 3: 刷新按钮不工作
**原因**: 初始化和刷新的逻辑有冲突
**解决方案**: 
- 初始化只在组件首次挂载时执行一次
- 刷新按钮明确触发新的绘制
- 风格变化自动重新生成

## 测试步骤

### 1. 初始加载测试
- 打开浏览器访问 http://localhost:3002
- 进入"幻想地图生成器"模块
- 验证地图一次性加载完成，不再闪烁

### 2. 地名多样性测试
- 查看地名列表
- 确认每个城市都有不同的名字
- 确认地名不会重复

### 3. 刷新功能测试
- 点击"重新生成地图"按钮
- 验证地图完全重绘（地形+城市）
- 验证新地图与之前明显不同

### 4. 风格切换测试
- 从下拉菜单选择不同风格（如"群岛模式"）
- 验证地图自动重新生成
- 验证地形特征符合所选风格

### 5. 稳定性测试
- 等待 30 秒不进行任何操作
- 验证地图保持静止，不再自动变化
- 验证城市位置保持不变

### 关键代码逻辑

1. **初始化控制**
```typescript

useEffect(() => {
  if (initializedRef.current) return;
  initializedRef.current = true;
  // 执行初始化绘制
```

## 已知限制

- 地图使用 Canvas 绘制，性能取决于浏览器
- 城市名称基于预设库生成，30个固定地名
- 地图风格有限：标准大陆、群岛、大陆、沙漠
